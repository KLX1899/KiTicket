# KiTicket documentation

This directory describes the code that is in this repository. It deliberately separates implemented behavior from future work, so it is safe to use as a development and review reference.

## Contents

- [Product vision](product/vision.md): users, scope and current product boundary.
- [Architecture overview](architecture/overview.md): runtime components, state and request flows.
- [Architecture decisions](architecture/decisions/): decisions that match the current implementation.
- [Diagrams](diagrams/): editable PlantUML models of the implemented system.
- [Requirements traceability](quality/requirements-traceability.md): feature-to-code and test mapping.
- [Implementation status](quality/implementation-status.md): known gaps and the next technical work.
- [Verification](quality/verification.md): repeatable local checks and their limits.
- [Local operations](operations/local-operations.md): startup, endpoints and troubleshooting.
- [Production readiness](operations/production-readiness.md): what the Kubernetes and Terraform material does—and does not—prove.
- [Backlog](planning/backlog.md): the prioritized follow-up work.

The root [README](../README.md) is the entry point for installing and running the project.
